# Movement Rules

Use these rules to decide where material belongs.

## Inbox
Put material in `dialogos/inbox/` when it is still raw.
Examples:
- a quote dump
- article notes
- transcript excerpts
- an unresolved hunch
- an initial question

## Forge
Move material to `dialogos/forge/` when active transformation begins.
Examples:
- argument maps
- rival framings
- pressure-tested distinctions
- compressed blueprints

## Syntheses
Move material to `dialogos/syntheses/` when it becomes strong enough to revisit later without redoing the whole conversation.
Examples:
- clear conceptual model
- durable essay-like note
- stable framework

## State
Use `dialogos/state/` to track the learner's current position, not to duplicate the full content.
Store:
- current phase
- current grip
- open loops
- linked artifacts
- next move

## memory/
Write to `memory/YYYY-MM-DD.md` only when the insight or decision should persist beyond the current thread.

## MEMORY.md
Promote only the most stable, reusable truths:
- enduring preferences
- lasting frameworks
- cross-project decisions
- important relationship context

## Rule of thumb
Recent is not the same as important.
Messy is not the same as useless.
Promote material based on durability and future retrieval value.
